Hydra - Getting Started

Obtaining the Hydra distribution

Source and binary versions of Hydra are available at http://www.cse.msu.edu/~acj/hydra.

Building Hydra

To build the source version of Hydra, unpack the distribution and then run make from the top-level directory.

If you downloaded the binary version, then you can skip to the next section.

Running Hydra and Performing Analysis

At its core, Hydra is a tool for transforming a UML behavioral model into a Promela model that can be formally analyzed. The Promela model can be used to verify that specific properties are true for every possible execution of the system described in the UML model. For example, we might want to ensure that the variable WidgetsAvailable is never negative or greater than 10.

In order to analyze a model, copy an XMI (XML Metadata Interchange) version of the model into the hydra directory. Most software modeling tools offer an option to export the current model into XMI.

Running Hydra with no arguments will produce the following output:

$ java -jar hydra.jar

Hydra v4.0

Usage: java -jar hydra.jar <model XMI input> <Promela output> [intermediate HIL output]

Hydra was developed by the Software Engineering and Network Systems Group
at Michigan State University.

If your XMI file is named MyModel.xmi, then the following command will tell Hydra to translate your model into Promela:

java -jar hydra.jar MyModel.xmi MyPromelaModel.pr

The last argument, MyPromelaModel.pr, is the Promela file that Hydra will produce.
