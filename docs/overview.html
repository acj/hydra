Hydra - Overview

MainDriver class

The MainDriver class takes care of bootstrapping and instrumenting Hydra. It parses the arguments that are passed on the command line and begins the process of parsing the XMI file being used as input.

h2PFoundation package

The h2PFoundation package contains foundation classes that support the translation of the Hydra Intermediate Language (HIL) into Promela. This includes utility classes for supporting parse tree nodes, as well as classes that define the HIL-to-Promela ("h2P") parser's symbol table.

h2PNodes package

The h2PNodes package contains a class for each of the nodes that can exist in a HIL parse tree.

h2PVisitors package

The h2PVisitors package contains classes that support the Visitor design pattern that is used to walk the HIL parse tree and perform pre- and post-processing steps on the tree (e.g., flattening composite states in some circumstances) to facilitate the translation of HIL into Promela.

The h2PVisitors.Parser sub-package contains the LL(k) grammars for the HIL parser and the UML expression parser.

umlModel package

The umlModel package contains classes that define a framework for representing a UML model. When XMI input models are parsed, a facsimile of the model is created using the classes in this package before the model is translated into HIL.

xmi2hil package

The xmi2hil package contains a set of classes that support the translation of XMI input data into the Hydra Intermediate Language (HIL). The ConversionDriver class is normally instantiated first and is responsible for instrumenting the translation process.

xmiParser package

The xmiParser package contains a set of classes that perform the parsing of XMI input data. They leverage the classes in umlModel and build a faithful model of the UML model stored in the XMI input data.
